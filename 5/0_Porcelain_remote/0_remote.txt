# 🔹 Advanced Remote Workflow in Git

## 1. Fork vs Clone
- **Fork (platform feature)**  
  - Happens on GitHub/GitLab.  
  - Creates a **remote copy** of someone else’s repo under your account.  
  - Used when you don’t have write access to the original repo.  
  - Example: Fork `original-owner/project` → becomes `your-username/project`.

- **Clone (Git command)**  
  - Happens locally on your machine.  
  - Creates a **local copy** of a remote repo.  
  - Sets up `origin` pointing to the repo you cloned.  
  - Example:  
    ```bash
    git clone https://github.com/your-username/project.git
    ```

---

## 2. Linking Local to Remote
- Start with a local repo:
  ```bash
  git init
  git remote add origin https://github.com/your-username/project.git
  git push -u origin main
  ```
- This pushes your local branch to the remote, creating `origin/main`.

---

## 3. Fetch vs Pull
- **Fetch**  
  - Downloads commits into tracking branches (`origin/main`).  
  - Does not change your local branch.  
  - Example:  
    ```bash
    git fetch origin
    git log HEAD..origin/main
    ```

- **Pull**  
  - Runs `fetch` + integrates changes.  
  - Default = merge, optional = rebase.  
  - Example:  
    ```bash
    git pull origin main        # merge
    git pull --rebase origin main  # rebase
    ```

---

## 4. Push
- Sends your local commits to the remote:
  ```bash
  git push origin main
  ```
- Updates `origin/main` with your local branch.

---

## 5. Upstream Workflow (Professional Contributor Cycle)
1. **Fork** the original repo on GitHub.  
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/project.git
   ```
3. **Add upstream** (original project):
   ```bash
   git remote add upstream https://github.com/original-owner/project.git
   ```
4. **Fetch upstream**:
   ```bash
   git fetch upstream
   ```
5. **Merge or rebase upstream changes**:
   ```bash
   git merge upstream/main
   # OR
   git rebase upstream/main
   ```
6. **Push to your fork**:
   ```bash
   git push origin main
   ```
7. **Open a Pull Request** → contribute back to upstream.

---

## 6. Advanced Remote Commands
- **Fetch all remotes**:
  ```bash
  git fetch --all
  ```
- **Prune deleted branches**:
  ```bash
  git fetch --prune
  ```
- **Change remote name**:
  ```bash
  git remote rename origin not_origin
  ```
- **View remotes**:
  ```bash
  git remote -v
  ```

---

## 7. Scenario Flow (Local ↔ Origin ↔ Upstream)

- **Clone public repo directly** → read-only, can’t push.  
- **Fork + clone your fork** → full control, can push.  
- **Add upstream** → keep your fork synced with original.  
- **Fetch** → preview changes.  
- **Pull** → integrate changes.  
- **Push** → send your commits upstream.  
- **PR** → propose changes to original project.

---

## ✅ Key Takeaways
- **Fork** = remote copy under your account (platform feature).  
- **Clone** = local copy on your machine (Git command).  
- **Fetch** = safe preview of remote changes.  
- **Pull** = fetch + merge/rebase into your branch.  
- **Push** = send local commits to remote.  
- **Upstream workflow** = fork → clone → add upstream → fetch → merge/rebase → push → PR.
