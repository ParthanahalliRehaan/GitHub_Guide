## đź”ą 1. What is `git fetch`?
- **Purpose**: Downloads commits, branches, and tags from a remote repository into your local `.git` database.  
- **Key point**: It does **not** merge or rebase automatically â€” it just updates your remote tracking branches.  
đź‘‰ Safe way to preview remote changes before integrating.

---

## đź”ą 2. Basic Usage
```bash
git fetch
```
- Fetches updates from the default remote (`origin`).  
- Updates remote tracking branches like `origin/main`.  
- Your local branch (`main`) stays unchanged until you merge or rebase.

---

## đź”ą 3. Common Variants

### Fetch Specific Remote
```bash
git fetch origin
```

### Fetch Specific Branch
```bash
git fetch origin feature-x
```
- Downloads only that branch from remote.

### Fetch All Remotes
```bash
git fetch --all
```
- Updates all remotes configured in the repo.

---

## đź”ą 4. Useful Flags

| Flag | Purpose | Example |
|------|---------|---------|
| `--all` | Fetch from all remotes | `git fetch --all` |
| `--prune` | Remove deleted remote branches locally | `git fetch --prune` |
| `--tags` | Fetch all tags | `git fetch --tags` |
| `--depth=N` | Shallow fetch (limit history) | `git fetch --depth=1` |
| `--dry-run` | Show what would be fetched, without downloading | `git fetch --dry-run` |
| `--force` | Force overwrite of local refs | `git fetch --force` |
| `--refmap` | Control mapping of remote refs | `git fetch --refmap=+refs/heads/*:refs/remotes/origin/*` |

---

## đź”ą 5. Examples

### Fetch + Prune
```bash
git fetch --prune
```
- Cleans up local references to branches deleted on remote.  
- Keeps your repo tidy.

### Fetch Tags
```bash
git fetch --tags
```
- Downloads all tags from remote.  
- Useful for release management.

### Shallow Fetch
```bash
git fetch --depth=1
```
- Downloads only the latest commit(s).  
- Saves bandwidth and time.  
- Common in CI/CD pipelines.

---

## đź”ą 6. Inspecting Fetched Changes

After fetching, you can inspect remote updates:
```bash
git log HEAD..origin/main
```
- Shows commits present in remote but not in local.  

Or compare branches:
```bash
git diff main origin/main
```
- Shows differences between local and remote branch.

## đź”ą 7. Advanced Workflows

### Fetch + Checkout Remote Branch
```bash
git fetch origin
git checkout -b feature-x origin/feature-x
```
- Creates a local branch tracking a remote branch.

### Fetch Specific Ref
```bash
git fetch origin refs/heads/feature-x:refs/remotes/origin/feature-x
```
- Fetches a single branch explicitly.

### Fetch with Force
```bash
git fetch --force origin main
```
- Overwrites local tracking branch with remote state.
