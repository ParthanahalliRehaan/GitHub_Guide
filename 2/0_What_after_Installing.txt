# 🧭 Git Config Deep Dive (V2 with Definitions & Purposes)

## 🔹 1. Git Config Structure
Git config files are **INI-style**:  
- **Section**: Groups of related settings (`[user]`, `[core]`, `[alias]`, etc.)  
- **Key**: Specific setting inside a section (`name`, `email`, `editor`)  
- **Value**: The actual data (`"Rehaan"`, `"vim"`, `"true"`)  

Example:
```ini
[user]                 # Section: identity
    name = Rehaan      # Key: name → Value: "Rehaan"
    email = rehaan@example.com

[core]                 # Section: core behavior
    editor = code --wait

[alias]                # Section: shortcuts
    st = status
    co = checkout
```

---

## 🔹 2. Basic Config (Identity & Essentials)
```bash
git config --global user.name "Rehaan"      # [user] name → who you are
git config --global user.email "rehaan@example.com"  # [user] email → used in commits
git config --global core.editor "code --wait"        # [core] editor → default editor
git config --list
```

**Purpose**:  
- `[user]` section defines your identity for commits.  
- `[core]` section controls fundamental Git behavior (editor, file handling).  

---

## 🔹 3. Scope & Options
| Scope   | Flag        | File location        | Purpose |
|---------|-------------|----------------------|---------|
| Local   | `--local`   | `.git/config`        | Repo-specific settings |
| Global  | `--global`  | `~/.gitconfig`       | User-wide defaults |
| System  | `--system`  | `/etc/gitconfig`     | System-wide defaults |

## 🔹 4. Advanced Config (Customization)

### Aliases (shortcuts)
```bash
git config --global alias.st status      # "st" → shortcut for `git status`
git config --global alias.co checkout    # "co" → shortcut for `git checkout`
git config --global alias.br branch      # "br" → shortcut for `git branch`
git config --global alias.lg "log --oneline --graph --decorate"
```

**Purpose of aliases**:  
- `st`: Quickly check repo status.  
- `co`: Shorter way to switch branches.  
- `br`: Manage branches faster.  
- `lg`: Visual log with graph, one-line commits, decorations.  

👉 Aliases save keystrokes and make daily Git usage smoother.

---

### Credentials
```bash
git config --global credential.helper cache       # cache credentials temporarily
git config --global credential.helper store       # store permanently (plaintext)
git config --global credential.helper manager-core # use Windows Credential Manager
```

**Purpose**: Avoid typing username/password every time you push/pull.

---

### Line Endings & File Handling
```bash
git config --global core.autocrlf true   # convert line endings safely (Windows)
git config --global core.fileMode false  # ignore file permission changes
```

**Purpose**: Prevent cross-platform issues with line endings and file modes.

---

### Commit Signing
```bash
git config --global commit.gpgsign true
git config --global user.signingkey <your-gpg-key-id>
```

**Purpose**: Sign commits with GPG for authenticity (used in open-source projects).

---

### Push & Pull Behavior
```bash
git config --global push.default simple   # safer push mode
git config --global pull.rebase true      # rebase instead of merge
```

**Purpose**: Control how branches sync with remotes.

---

## 🔹 5. Append, Replace, Delete Variations
### Add (append multiple values)
```bash
git config --global --add user.email "alt@example.com"
git config --global --add remote.origin.pushurl "git@github.com:rehaan/project.git"
```

**Purpose**: Allow multiple values (e.g., multiple emails or push URLs).

---

### Replace (default)
```bash
git config --global user.email "new@example.com"
```

**Purpose**: Overwrite existing value.

---

### Delete
```bash
git config --global --unset user.email        # remove one value
git config --global --unset-all user.email    # remove all values
git config --global --unset alias.st          # remove alias
```

**Purpose**: Clean up or reset unwanted config.

---

### Inspect/Edit
```bash
git config --global user.name                 # show specific key
git config --global --edit                    # edit global config file
git config --system --edit                    # edit system config
git config --local --edit                     # edit repo config
```

**Purpose**: Inspect or manually edit config files.

## 🔹 7. Pro-Level Config
- **Conditional Includes**  
  ```ini
  [includeIf "gitdir:~/work/"]
      path = ~/.gitconfig-work
  ```
  *Purpose*: Different configs for work vs personal repos.

- **Per-Repo Identity**  
  ```bash
  git config user.name "WorkUser"
  git config user.email "work@example.com"
  ```
  *Purpose*: Use different identities depending on repo.

- **Custom Tools**  
  ```bash
  git config --global merge.tool vimdiff
  git config --global diff.tool meld
  ```
  *Purpose*: Define merge/diff tools for conflict resolution.
