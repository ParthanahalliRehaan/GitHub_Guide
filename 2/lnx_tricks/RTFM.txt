## рЯУЦ RTFM вАУ *Read the Friendly Manual*
- **`man xyz`** вЖТ Opens the manual page for the command `xyz`.  
- **`man git` vs `man git-xyz`** вЖТ  
  - `man git` shows the general Git manual.  
  - `man git-commit`, `man git-push`, etc. show manuals for specific Git subcommands.  
- **Convention**:  
  - `man [section] keyword` вЖТ Section numbers (1вАУ8) categorize manuals (commands, system calls, configs, etc.).  
  - Example: `man 1 ls` (user command), `man 2 open` (system call).

## вМ®пЄП Navigation Inside `man` Pages
When you open a manual page, youвАЩre inside a pager (usually `less`). These are the key bindings:

| Key | Action |
|-----|--------|
| `j` / `k` | Scroll down / up one line |
| `J` / `K` | Scroll down / up half a page |
| `n` / `N` | Jump to next / previous search result |
| `/keyword` | Search forward for вАЬkeywordвАЭ |
| `?keyword` | Search backward for вАЬkeywordвАЭ |
| `h` | Help screen |
| `q` | Quit |

рЯСЙ Example: Inside `man git`, type `/commit` to jump to the section about commits. Use `n` to move to the next occurrence.

## рЯЫ†пЄП Alternative to `man`
Sometimes commands donвАЩt have man pages, or you want quick help:

- **`command --help`** вЖТ Shows usage and options.  
  Example:  
  ```bash
  id --help
  ```
  gives a summary of options for the `id` command.  
- Short forms like `-h` often work too:  
  ```bash

  ```

## рЯІ© Putting It All Together
1. **General command help** вЖТ `man xyz` or `xyz --help`.  
2. **Git-specific help** вЖТ `man git-xyz` or `git help xyz`.  
3. **Navigation inside manuals** вЖТ `j/k` for lines, `J/K` for half-pages, `/` and `?` for search, `n/N` to cycle results.  
4. **Fallback** вЖТ Online docs (git-scm.com, GNU manuals, etc.).
