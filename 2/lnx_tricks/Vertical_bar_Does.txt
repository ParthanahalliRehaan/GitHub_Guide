The **vertical bar (`|`)** in commands is most commonly used as a **pipe operator** in Unix-like shells (like Bash, Zsh, etc.).  

### 🔧 What it does
- It **takes the output of one command** and **feeds it as input to another command**.
- This allows you to chain commands together, creating powerful workflows without temporary files.

### 📌 Example
```bash
ls -l | grep ".txt"
```
- `ls -l` lists files in long format.
- The `|` sends that list directly into `grep ".txt"`.
- `grep` then filters only the files ending with `.txt`.

### 🧩 Why it’s useful
- **Efficiency**: You don’t need to save intermediate results.
- **Modularity**: Each command does one job well, and pipes let you combine them.
- **Readability**: It’s easier to understand a pipeline than a huge monolithic command.

### ⚡ Other uses of `|`
- In some programming languages (like C, JavaScript, Python), `|` is used for **bitwise OR**.
- In regex, `|` means **OR** (e.g., `cat|dog` matches either "cat" or "dog").
- In command-line tools like `awk` or `sed`, it can appear inside patterns or expressions.

👉 So, in shell commands, think of `|` as a **connector** that lets commands "talk" to each other.
