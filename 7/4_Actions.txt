## ⚙️ Purpose of Actions
- GitHub Actions lets you **automate workflows** directly in your repo.  
- Common uses: testing code, building apps, deploying to servers, publishing packages.  
- Workflows are defined in **YAML files** inside `.github/workflows/`.

---

## 🛠️ 1. Basic Workflow Setup

**Scenario:** You want to run tests automatically when code is pushed.

- Create file: `.github/workflows/test.yml`
```yaml
name: Run Tests
on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm install
      - name: Run tests
        run: npm test
```

- **Output (in GitHub UI):**
  ```
  ✓ Workflow "Run Tests" triggered by push
  ✓ Job test completed successfully
  ```

## 🚀 2. Deployment Workflow

**Scenario:** You want to deploy to GitHub Pages when pushing to `main`.

- File: `.github/workflows/deploy.yml`
```yaml
name: Deploy to Pages
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install && npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

- **Output:**
  ```
  ✓ Build completed
  ✓ Deployed site to GitHub Pages
  ```

## 🧩 3. Matrix Testing

**Scenario:** You want to test your app on multiple Node.js versions.

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node: [16, 18, 20]
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: ${{ matrix.node }}
      - run: npm install && npm test
```

- **Output:**
  ```
  ✓ Tests passed on Node 16
  ✓ Tests passed on Node 18
  ✓ Tests passed on Node 20
  ```

## 🔐 4. Secrets & Environment Variables

**Scenario:** You want to deploy with an API key.

- Store secret in repo: `API_KEY`.  
- Use in workflow:
```yaml
- name: Deploy
  run: node deploy.js
  env:
    API_KEY: ${{ secrets.API_KEY }}
```

- **Output:**
  ```
  Deploy script executed with API_KEY
  ```

---

## 📊 5. Monitoring & Logs

- **Check workflow runs:**
  ```bash
  gh run list
  ```
  Output:
  ```
  Run ID 101  Run Tests   Success
  Run ID 102  Deploy      Failed
  ```

- **View logs:**
  ```bash
  gh run view 102 --log
  ```
  Output:
  ```
  Error: Missing API_KEY
  ```

---

## ⚖️ Summary
- **Actions = automation engine.**  
- Define workflows in YAML → triggered by events (push, PR, schedule).  
- Use jobs, steps, secrets, and matrix strategies to build flexible pipelines.  
- They complement repos (code), issues (tasks), projects (planning), and packages (distribution).
