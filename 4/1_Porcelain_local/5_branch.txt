# 🧭 Git Branching Deep Dive (V2 with Definitions & Purposes)

## 🔹 1. Basic Branch Concepts
- A **branch** is just a pointer (reference) to a commit.  
- Default branch is usually `main` (or `master` in older repos).  
- Branches let you work on features or fixes without disturbing the main line of development.

---

## 🔹 2. `git branch` — Managing Branches

### Basic Usage
```bash
git branch              # list branches
git branch feature-x    # create new branch
git branch -d feature-x # delete branch (safe, only if merged)
git branch -D feature-x # force delete branch
```

### Options
| Flag | Purpose | Example |
|------|---------|---------|
| `-a` | List all branches (local + remote) | `git branch -a` |
| `-r` | List remote branches only | `git branch -r` |
| `-m` | Rename branch | `git branch -m old new` |
| `-vv` | Show tracking info | `git branch -vv` |

---

## 🔹 3. `git checkout` vs `git switch`

### `git checkout`
```bash
git checkout feature-x   # switch to branch
git checkout -b new-branch # create + switch
git checkout commitID file.txt # restore file from commit
```
- Versatile but overloaded (branch switching + file restoration).

### `git switch` (newer, clearer)
```bash
git switch feature-x       # switch branch
git switch -c new-branch   # create + switch
```
- Dedicated to branch operations.  
- Easier to understand than `checkout`.

---

## 🔹 4. Branch Tracking & Upstream

### Set Upstream Branch
```bash
git branch --set-upstream-to=origin/main
```
- Links local branch to remote branch.  
- Enables `git pull`/`git push` without specifying remote.

### Show Tracking Info
```bash
git branch -vv
```
- Displays which remote branch each local branch tracks.

---

## 🔹 5. Merging Branches

### Fast-Forward Merge
```bash
git merge feature-x
```
- If no new commits on target branch, Git just moves the pointer forward.  
- No merge commit created.

### No Fast-Forward Merge
```bash
git merge --no-ff feature-x
```
- Forces a merge commit even if fast-forward is possible.  
- Preserves branch history explicitly.

### Squash Merge
```bash
git merge --squash feature-x
git commit -m "Squash feature-x"
```
- Combines all commits from branch into one commit.  
- Useful for keeping history clean.

---

## 🔹 6. Advanced Branching Workflows

### Create Branch from Specific Commit
```bash
git branch feature-x commitID
```

### Track Remote Branch Automatically
```bash
git checkout -b feature-x origin/feature-x
```
or
```bash
git switch -c feature-x --track origin/feature-x
```

### Delete Remote Branch
```bash
git push origin --delete feature-x
```

---

## 🔹 7. Summary Table

| Command | Purpose |
|---------|---------|
| `git branch` | List/create/delete branches |
| `git branch -a` | Show local + remote branches |
| `git branch -m old new` | Rename branch |
| `git checkout branch` | Switch branch (legacy) |
| `git checkout -b new` | Create + switch branch |
| `git switch branch` | Switch branch (modern) |
| `git switch -c new` | Create + switch branch |
| `git merge branch` | Merge branch (fast-forward if possible) |
| `git merge --no-ff branch` | Force merge commit |
| `git merge --squash branch` | Squash commits into one |
| `git branch -vv` | Show tracking info |
