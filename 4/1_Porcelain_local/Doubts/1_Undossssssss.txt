## 1. Undo with `git reset`
`reset` moves the **HEAD pointer** and optionally changes the staging area/working directory.

- **Undo commit, keep changes staged**  
  ```bash
  git reset --soft HEAD~1
  ```
  вЖТ Removes last commit but keeps changes staged.  
  рЯСЙ Best if you just want to rewrite the commit message.

- **Undo commit, keep changes unstaged**  
  ```bash
  git reset --mixed HEAD~1
  ```
  вЖТ Removes last commit, keeps changes in working directory but unstaged.  
  рЯСЙ Default mode.

- **Undo commit, discard changes**  
  ```bash
  git reset --hard HEAD~1
  ```
  вЖТ Removes last commit and discards changes.  
  вЪ†пЄП Dangerous вАФ data lost unless recovered via `reflog`.

## 2. Undo with `git revert`
`revert` creates a **new commit** that undoes the changes of a previous commit.

- **Revert a single commit**  
  ```bash
  git revert <commit-id>
  ```
  вЖТ Safe for shared repos because history isnвАЩt rewritten.

- **Revert a merge commit**  
  ```bash
  git revert -m 1 <merge-commit-id>
  ```
  вЖТ Undoes a merge, keeping parent branch specified by `-m`.

рЯСЙ Use `revert` when commits are already pushed and shared.

---

## 3. Undo with `git restore`
`restore` is a newer command for undoing file changes.

- **Discard changes in working directory**  
  ```bash
  git restore file.txt
  ```
  вЖТ Restores file to last committed state.

- **Unstage a file (undo add)**  
  ```bash
  git restore --staged file.txt
  ```
  вЖТ Removes file from staging area without discarding edits.

рЯСЙ Use `restore` for file-level undo (working directory or staging).

---

## 4. Undo with `git reflog`
`reflog` tracks all HEAD movements (commits, resets, checkouts).

- **View HEAD history**  
  ```bash
  git reflog
  ```
  Output example:
  ```
  a1b2c3 HEAD@{0}: commit: Add login
  d4e5f6 HEAD@{1}: reset: moving to HEAD~1
  ```
- **Recover lost commit**  
  ```bash
  git checkout a1b2c3
  ```
  вЖТ Lets you restore commits even after `reset --hard`.

рЯСЙ Use `reflog` as a safety net when you think youвАЩve вАЬlostвАЭ commits.

## 5. Comparison Table

| Tool | Purpose | Safe for Shared History? |
|------|---------|---------------------------|
| `reset --soft` | Undo commit, keep staged | вЭМ Rewrites history |
| `reset --mixed` | Undo commit, keep unstaged | вЭМ Rewrites history |
| `reset --hard` | Undo commit, discard changes | вЭМ Rewrites history |

## 6. Practical Scenarios

- **Wrong commit message** вЖТ `git commit --amend` or `git reset --soft HEAD~1` then recommit.  
- **Committed too early** вЖТ `git reset --mixed HEAD~1` (keep changes unstaged).  
- **Want to erase commit completely** вЖТ `git reset --hard HEAD~1` (вЪ†пЄП careful).  
- **Undo commit already pushed** вЖТ `git revert <commit-id>` (safe).  
- **Accidentally staged wrong file** вЖТ `git restore --staged file.txt`.  
- **Lost commit after reset** вЖТ `git reflog` to find SHA, then `git checkout <sha>`.
