

## ⚔️ What is a Merge Conflict?
A merge conflict happens when Git cannot automatically reconcile differences between two commits. Advanced conflicts often occur when:
- Multiple developers edit the same lines.
- File renames/moves overlap with edits.
- Rebase replay introduces conflicts across several commits.

---

## 🛠 Advanced Conflict Resolution Workflow

### 1. Detect Conflicts
```bash
git merge main
# or during rebase
git rebase main
```
Git will pause and mark conflicts in files with:
```text
<<<<<<< HEAD
your changes
=======
incoming changes
>>>>>>> main
```

---

### 2. Use `git status` to see conflict map
```bash
git status
```
Shows which files are **unmerged**. This is your checklist.

---

### 3. Resolve Conflicts Strategically
- **Manual edit:** Open file, decide which code to keep.
- **Three-way merge tool:**  
  ```bash
  git mergetool
  ```
  Launches a GUI (like `vimdiff`, `meld`, or VSCode merge editor).
- **Preserve both changes:** Sometimes you want both versions:
  ```text
  <<<<<<< HEAD
  your code
  =======
  their code
  >>>>>>> main
  ```
  → Replace with a merged version that integrates both.

---

### 4. Stage Resolved Files
```bash
git add conflicted_file.js
```

---

### 5. Continue Workflow
- If merging:
  ```bash
  git commit
  ```
- If rebasing:
  ```bash
  git rebase --continue
  ```

---

## ⚡ Advanced Techniques

| Technique | Command | Use Case |
|-----------|---------|----------|
| **Abort merge/rebase** | `git merge --abort` / `git rebase --abort` | When conflicts are too messy, restart clean |
| **Skip commit in rebase** | `git rebase --skip` | If a commit is obsolete or unnecessary |
| **Interactive rebase** | `git rebase -i main` | Reorder, squash, or drop commits while resolving |
| **Conflict markers diff** | `git diff` | See exactly what changed before resolving |
| **Ours vs Theirs strategy** | `git checkout --ours file.txt` / `git checkout --theirs file.txt` | Accept one side wholesale |

---

## 🧠 Strategic Analogy
Think of conflict resolution like **merging two recipes**:
- Merge = keep both recipes intact, add a note where they diverged.
- Rebase = rewrite your recipe so it looks like you always cooked after theirs.
- Advanced resolution = deciding whether to keep your spice mix, theirs, or invent a hybrid dish.

---

## 🚀 Pro Tip
For large teams:
- Use **rebase before PR** to minimize conflicts.
- Use **merge for integration** to preserve history.
- Automate conflict detection with CI (e.g., GitHub Actions running `git merge --no-commit --no-ff`).
